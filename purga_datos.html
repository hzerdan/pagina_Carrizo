Arquitectura de Purga de Datos

El sistema almacena información en Supabase organizada en dos tipos de entidades. Es fundamental mantener esta distinción para no romper la operatividad técnica de la plataforma:

🗑️

Datos Transaccionales

Información generada por el flujo diario de los usuarios. Se puede purgar según fecha límite.

  • Pedidos (y sus instancias)
  • Órdenes de Compra (e instancias)
  • Remitos (e ítems)
  • Inspecciones e imágenes
  • Mensajes y chats de soporte
  • Historiales y logs técnicos
🔒

Datos Maestros (Configuración)

Definiciones estructurales del negocio y del sistema. No se tocan bajo ninguna circunstancia.

  • Definiciones de estados FSM
  • Configuración de validaciones
  • Roles del sistema
  • Camiones, Choferes y Transportistas
  • Usuarios y perfiles
💡

Regla de Protección Retroactiva

Los usuarios cargan órdenes hoy que corresponden a transacciones comerciales del pasado. Por lo tanto, el sistema de purga evalúa que tanto la fecha comercial del documento como la fecha física de creación del registro en la base de datos (created_at) sean anteriores a la fecha límite para proceder a la eliminación.

Flujo de Operación Correcto

Para realizar la purga sin dejar registros huérfanos ni archivos basura en la nube, se debe seguir estrictamente este orden:

  1. Ejecutar la consulta del Paso 1 (Storage) para generar la lista de archivos a borrar.
  2. Exportar los resultados como CSV, renombrarlo a files_to_delete.csv y copiarlo a la raíz del proyecto.
  3. Ejecutar el archivo automatizado delete_storage_files.bat para limpiar físicamente el Storage.
  4. Ejecutar el script del Paso 2 (SQL Purge) en el panel de Supabase para limpiar la base de datos pública y actualizar las vistas.

Paso 1: Limpieza de Archivos en Supabase Storage

⚠️

¿Por qué se requiere un script externo?

La base de datos de Supabase protege los metadatos de almacenamiento mediante el trigger storage.protect_delete(). Ejecutar un DELETE directo en SQL fallará con un error 42501 y, además, dejaría los archivos físicos huérfanos en los servidores en la nube. Debemos borrarlos a través de la API oficial.

1. Generar el Reporte CSV

Ejecute esta consulta en el **SQL Editor** de Supabase para obtener todas las rutas físicas de archivos a eliminar:

sql (Consulta de Archivos a Borrar) 
-- Consulta de archivos a eliminar
WITH vars AS (
    SELECT 
        '2026-05-20 00:00:00-03'::timestamptz AS cutoff,
        '2026-05-20'::date AS cutoff_date
)
-- 1. Archivos en public.documentos
SELECT 
    COALESCE(
        substring(storage_path from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/'),
        'Archivos_entrada'
    ) AS bucket_id,
    CASE 
        WHEN storage_path LIKE '%/storage/v1/object/%' THEN 
            substring(storage_path from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$')
        ELSE 
            storage_path
    END AS file_path
FROM public.documentos, vars
WHERE uploaded_at < vars.cutoff
   OR pedido_instance_id IN (SELECT id FROM public.pedido_instancias WHERE created_at < vars.cutoff)
   OR oc_instance_id IN (SELECT id FROM public.oc_instancias WHERE created_at < vars.cutoff)
   OR remito_id IN (SELECT id FROM public.remitos WHERE created_at < vars.cutoff)

UNION ALL

-- 2. Archivos multimedia de chats (evidencias)
SELECT 
    storage_bucket AS bucket_id,
    storage_path AS file_path
FROM public.message_media, vars
WHERE created_at < vars.cutoff

UNION ALL

-- 3. PDFs/Archivos adjuntos en remitos
SELECT 
    substring(archivo_url from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/') AS bucket_id,
    substring(archivo_url from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$') AS file_path
FROM public.remitos, vars
WHERE created_at < vars.cutoff AND archivo_url IS NOT NULL AND archivo_url LIKE '%/storage/v1/object/%'

UNION ALL

SELECT 
    substring(cotizacion_url from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/') AS bucket_id,
    substring(cotizacion_url from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$') AS file_path
FROM public.remitos, vars
WHERE created_at < vars.cutoff AND cotizacion_url IS NOT NULL AND cotizacion_url LIKE '%/storage/v1/object/%'

UNION ALL

-- 4. Planillas de inspecciones
SELECT 
    substring(planilla_url from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/') AS bucket_id,
    substring(planilla_url from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$') AS file_path
FROM public.inspecciones, vars
WHERE (created_at < vars.cutoff OR remito_id IN (SELECT id FROM public.remitos WHERE created_at < vars.cutoff))
  AND planilla_url IS NOT NULL AND planilla_url LIKE '%/storage/v1/object/%'

UNION ALL

SELECT 
    substring(planilla_personalizada_url from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/') AS bucket_id,
    substring(planilla_personalizada_url from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$') AS file_path
FROM public.inspecciones, vars
WHERE (created_at < vars.cutoff OR remito_id IN (SELECT id FROM public.remitos WHERE created_at < vars.cutoff))
  AND planilla_personalizada_url IS NOT NULL AND planilla_personalizada_url LIKE '%/storage/v1/object/%'

UNION ALL

SELECT 
    substring(planilla_completada_url from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/') AS bucket_id,
    substring(planilla_completada_url from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$') AS file_path
FROM public.inspecciones, vars
WHERE (created_at < vars.cutoff OR remito_id IN (SELECT id FROM public.remitos WHERE created_at < vars.cutoff))
  AND planilla_completada_url IS NOT NULL AND planilla_completada_url LIKE '%/storage/v1/object/%'

UNION ALL

-- 5. Cotizaciones de pedidos
SELECT 
    substring(cotizacion_archivo_url from '/storage/v1/object/(?:public|authenticated|sign)/([^/]+)/') AS bucket_id,
    substring(cotizacion_archivo_url from '/storage/v1/object/(?:public|authenticated|sign)/[^/]+/(.+)$') AS file_path
FROM public.pedidos, vars
WHERE (fecha_pedido < vars.cutoff_date OR (fecha_pedido IS NULL AND created_at < vars.cutoff))
  AND cotizacion_archivo_url IS NOT NULL AND cotizacion_archivo_url LIKE '%/storage/v1/object/%';

2. Ejecutar la Eliminación Masiva

  • Descargue la tabla generada en el paso anterior como un archivo **CSV** desde la consola de Supabase.
  • Guarde el archivo con el nombre exacto `files_to_delete.csv` en la carpeta raíz del proyecto.
  • Haga doble clic en el archivo ejecutable delete_storage_files.bat.
  • Cuando la consola se lo solicite, pegue la **SERVICE_ROLE_KEY** de su proyecto y pulse Enter. El script procesará y eliminará todos los archivos en lotes automáticos de hasta 1000 elementos.
🛠️

Scripts de Soporte Locales

Los scripts encargados de esta operación son:
• Script principal: delete_storage_files.ps1 (PowerShell)
• Acceso directo: delete_storage_files.bat (Consola Windows)

Paso 2: Script SQL de Purga en la Base de Datos

Ejecute este script en el SQL Editor de Supabase. El script realiza la desvinculación de claves recursivas en tablas jerárquicas y luego procede a una eliminación bottom-up segura para cumplir con las restricciones relacionales.

sql (Script de Purga Definitivo) 
DO $$
DECLARE
    -- MODIFIQUE AQUÍ LA FECHA LÍMITE DE BORRADO:
    _cutoff TIMESTAMPTZ := '2026-05-20 00:00:00-03'; -- Timestamp con zona horaria
    _cutoff_date DATE   := '2026-05-20';             -- Fecha para filtros date
BEGIN
    RAISE NOTICE 'Iniciando purga de datos transaccionales en base de datos anteriores a %...', _cutoff;

    -- =========================================================================
    -- 1. PRE-CALCULAR IDS DE ENTIDADES A BORRAR EN TABLAS TEMPORALES
    -- =========================================================================
    CREATE TEMP TABLE tmp_pedidos_to_delete AS
    SELECT id FROM public.pedidos 
    WHERE (fecha_pedido < _cutoff_date OR fecha_pedido IS NULL)
      AND created_at < _cutoff;

    CREATE TEMP TABLE tmp_ocs_to_delete AS
    SELECT id FROM public.ordenes_compra 
    WHERE (fecha_emision < _cutoff_date OR fecha_emision IS NULL)
      AND created_at < _cutoff;

    CREATE TEMP TABLE tmp_pedido_instances_to_delete AS
    SELECT id FROM public.pedido_instancias 
    WHERE created_at < _cutoff 
       OR pedido_id IN (SELECT id FROM tmp_pedidos_to_delete);

    CREATE TEMP TABLE tmp_oc_instances_to_delete AS
    SELECT id FROM public.oc_instancias 
    WHERE created_at < _cutoff 
       OR oc_id IN (SELECT id FROM tmp_ocs_to_delete);

    CREATE TEMP TABLE tmp_remitos_to_delete AS
    SELECT id FROM public.remitos 
    WHERE created_at < _cutoff
       OR id IN (
           SELECT DISTINCT remito_id FROM public.remito_items 
           WHERE origen_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
              OR destino_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
              OR origen_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete)
              OR destino_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete)
       );

    CREATE TEMP TABLE tmp_inspecciones_to_delete AS
    SELECT id FROM public.inspecciones 
    WHERE created_at < _cutoff 
       OR remito_id IN (SELECT id FROM tmp_remitos_to_delete);

    CREATE TEMP TABLE tmp_conversations_to_delete AS
    SELECT id FROM public.conversations 
    WHERE created_at < _cutoff
       OR remito_actual_id IN (SELECT id FROM tmp_remitos_to_delete);

    -- =========================================================================
    -- 2. DESVINCULAR RELACIONES RECURSIVAS EN LA BASE DE DATOS
    -- =========================================================================
    UPDATE public.pedido_instancias 
    SET parent_instance_id = NULL 
    WHERE parent_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR id IN (SELECT id FROM tmp_pedido_instances_to_delete);

    UPDATE public.oc_instancias 
    SET parent_instance_id = NULL 
    WHERE parent_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete)
       OR id IN (SELECT id FROM tmp_oc_instances_to_delete);

    UPDATE public.remitos 
    SET parent_remito_id = NULL 
    WHERE parent_remito_id IN (SELECT id FROM tmp_remitos_to_delete)
       OR id IN (SELECT id FROM tmp_remitos_to_delete);

    -- =========================================================================
    -- 3. ELIMINAR TABLAS DE LOGS, AUDITORÍA Y METADATOS TRANSACCIONALES
    -- =========================================================================
    DELETE FROM public.message_media
    WHERE created_at < _cutoff 
       OR message_id IN (
           SELECT id FROM public.conversation_messages 
           WHERE conversation_id IN (SELECT id FROM tmp_conversations_to_delete)
       );

    DELETE FROM public.conversation_messages
    WHERE created_at < _cutoff 
       OR conversation_id IN (SELECT id FROM tmp_conversations_to_delete);

    DELETE FROM public.conversations
    WHERE id IN (SELECT id FROM tmp_conversations_to_delete);

    DELETE FROM public.alertas_escalamiento
    WHERE created_at < _cutoff;

    DELETE FROM public.magic_links
    WHERE created_at < _cutoff;

    DELETE FROM public.debug_log_fsm
    WHERE timestamp < _cutoff;

    DELETE FROM public.mv_refresh_events
    WHERE refreshed_at < _cutoff;

    DELETE FROM public.documentos
    WHERE uploaded_at < _cutoff
       OR pedido_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR oc_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete)
       OR remito_id IN (SELECT id FROM tmp_remitos_to_delete);

    DELETE FROM public.historial_eventos
    WHERE timestamp < _cutoff
       OR pedido_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR oc_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete);

    -- =========================================================================
    -- 4. ELIMINAR VINCULACIONES E ÍTEMS TRANSACCIONALES
    -- =========================================================================
    DELETE FROM public.vinculaciones_pedido_oc
    WHERE pedido_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR oc_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete);

    DELETE FROM public.inspeccion_items_pedido
    WHERE pedido_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR inspeccion_id IN (SELECT id FROM tmp_inspecciones_to_delete);

    DELETE FROM public.inspecciones
    WHERE id IN (SELECT id FROM tmp_inspecciones_to_delete);

    DELETE FROM public.remito_items
    WHERE remito_id IN (SELECT id FROM tmp_remitos_to_delete)
       OR origen_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR destino_instance_id IN (SELECT id FROM tmp_pedido_instances_to_delete)
       OR origen_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete)
       OR destino_instance_id IN (SELECT id FROM tmp_oc_instances_to_delete);

    -- =========================================================================
    -- 5. ELIMINAR ENTIDADES TRANSACCIONALES PRINCIPALES
    -- =========================================================================
    DELETE FROM public.remitos
    WHERE id IN (SELECT id FROM tmp_remitos_to_delete);

    DELETE FROM public.pedido_instancias
    WHERE id IN (SELECT id FROM tmp_pedido_instances_to_delete);

    DELETE FROM public.oc_instancias
    WHERE id IN (SELECT id FROM tmp_oc_instances_to_delete);

    DELETE FROM public.pedidos
    WHERE id IN (SELECT id FROM tmp_pedidos_to_delete);

    DELETE FROM public.ordenes_compra
    WHERE id IN (SELECT id FROM tmp_ocs_to_delete);

    -- =========================================================================
    -- 6. REFRESCAR VISTAS MATERIALIZADAS
    -- =========================================================================
    RAISE NOTICE 'Refrescando vistas materializadas...';
    REFRESH MATERIALIZED VIEW public.mv_analisis_logistico;
    REFRESH MATERIALIZED VIEW public.mv_detalle_instancias_activas;
    REFRESH MATERIALIZED VIEW public.mv_instancias_analitica;
    REFRESH MATERIALIZED VIEW public.mv_instancias_por_estado;
    REFRESH MATERIALIZED VIEW public.mv_tiempo_promedio_estado;
    REFRESH MATERIALIZED VIEW public.mv_tiempo_promedio_por_resultado;

    -- Limpiar tablas temporales
    DROP TABLE tmp_pedidos_to_delete;
    DROP TABLE tmp_ocs_to_delete;
    DROP TABLE tmp_pedido_instances_to_delete;
    DROP TABLE tmp_oc_instances_to_delete;
    DROP TABLE tmp_remitos_to_delete;
    DROP TABLE tmp_inspecciones_to_delete;
    DROP TABLE tmp_conversations_to_delete;

    RAISE NOTICE 'Purga relacional completa y exitosa.';
END $$;

Simulador de Purga (Dry Run / Prueba en Seco)

¡RECOMENDADO ANTES DE CADA PURGA! Ejecute esta consulta en el editor SQL para auditar exactamente qué registros se eliminarían y cuáles se conservarían según la fecha límite configurada.

sql (Simulador de Auditoría) 
-- Simulador de Purga
WITH vars AS (
    SELECT 
        '2026-05-20 00:00:00-03'::timestamptz AS cutoff,
        '2026-05-20'::date AS cutoff_date
),
pedidos_eval AS (
    SELECT 
        id, 
        fecha_pedido AS fecha_referencia, 
        created_at, 
        pedido_ref_externa AS referencia_externa,
        'Pedido' AS tipo_entidad,
        CASE 
            WHEN (fecha_pedido < vars.cutoff_date OR fecha_pedido IS NULL) AND created_at < vars.cutoff THEN 'ELIMINAR'
            ELSE 'CONSERVAR'
        END AS accion
    FROM public.pedidos, vars
),
ocs_eval AS (
    SELECT 
        id, 
        fecha_emision AS fecha_referencia, 
        created_at, 
        oc_ref_externa AS referencia_externa,
        'Orden de Compra' AS tipo_entidad,
        CASE 
            WHEN (fecha_emision < vars.cutoff_date OR fecha_emision IS NULL) AND created_at < vars.cutoff THEN 'ELIMINAR'
            ELSE 'CONSERVAR'
        END AS accion
    FROM public.ordenes_compra, vars
),
remitos_eval AS (
    SELECT 
        r.id, 
        r.created_at::date AS fecha_referencia, 
        r.created_at, 
        r.numero_remito AS referencia_externa,
        'Remito' AS tipo_entidad,
        CASE 
            -- Un remito se borra si su fecha es anterior a cutoff O si pertenece a instancias borradas
            WHEN r.created_at < vars.cutoff OR r.id IN (
                SELECT DISTINCT remito_id FROM public.remito_items 
                WHERE origen_instance_id IN (SELECT id FROM public.pedido_instancias WHERE created_at < vars.cutoff OR pedido_id IN (SELECT id FROM public.pedidos WHERE (fecha_pedido < vars.cutoff_date OR fecha_pedido IS NULL) AND created_at < vars.cutoff))
                   OR destino_instance_id IN (SELECT id FROM public.pedido_instancias WHERE created_at < vars.cutoff OR pedido_id IN (SELECT id FROM public.pedidos WHERE (fecha_pedido < vars.cutoff_date OR fecha_pedido IS NULL) AND created_at < vars.cutoff))
            ) THEN 'ELIMINAR'
            ELSE 'CONSERVAR'
        END AS accion
    FROM public.remitos r, vars
)
SELECT accion, tipo_entidad, id, fecha_referencia, created_at, referencia_externa FROM pedidos_eval
UNION ALL
SELECT accion, tipo_entidad, id, fecha_referencia, created_at, referencia_externa FROM ocs_eval
UNION ALL
SELECT accion, tipo_entidad, id, fecha_referencia, created_at, referencia_externa FROM remitos_eval
ORDER BY accion DESC, tipo_entidad, id;
